本文用于记录本次博客搭建的全过程，以及在这个过程中我踩的一些坑。由于技术水平有限（我是菜鸡），难免会有一些错误（大佬轻拍），还请大佬指出，以便修改。

HEXO 博客搭建小白教程（一）：建站

前言

大学毕业之际，即将步入社会，心里却一片茫然，丝毫不知道将来要何去何从。这时想起了我那个夭折的 WordPress 博客，就想着要不要再次把自己的博客搭建起来，也好作为自己记录日常生活的一个位置。

说干就干！鉴于上次购买云服务器搭建博客，导致每个月都要花一笔钱续费却不怎么用的情况，这次决定换一种方式，寻找一个免费的平台。在 Google 上搜了一下，发现基于 Hexo+GitHub Pages 搭建的博客似乎挺适合我现在的状态。Markdown 语法格式，本地生成后直接推送 Github 无需额外服务器，要说缺点的话大概就是 Github 服务器在海外，国内的连接速度会比较慢。不过可以使用一些 CDN 进行加速，也能提高一些国内的访问速度。

安装 Node.js

Hexo 是一个快速、简洁且高效的博客框架，具有超快速度、支持 Markdown、一键部署的优势。Hexo 基于 Node.js，所以首先要确保自己的电脑已经安装 Node.js。

Node.js 为大多数平台提供了官方的 安装程序。对于中国大陆地区用户，可以前往 淘宝 Node.js 镜像 下载。只需选择自己电脑的操作系统，按照安装提示一步一步操作，即可完成安装。

安装完成后，可以打开命令提示符，输入 node -v 和 npm -v，如果出现版本号，则说明 Node.js 安装成功。

对于中国大陆地区用户，可以在安装完成后进行淘宝镜像源加速，或为 npm 配置代理。

安装 Hexo

Node.js 安装完成后，即可使用 npm 安装 Hexo。安装完成后，输入 hexo -v，如果出现版本信息，则说明 Hexo 安装成功。

创建 GitHub Pages

首先需要注册一个 Github 账号，在 注册页面 输入用户名、邮箱、密码即可完成注册。注册完成后，在 登录页面 输入注册时使用的用户名和密码进行登录。登录完成后在右上角点击 New repository，新建一个项目库。

如下图所示，项目名称为自己的 用户名+.github.io 后缀，比如你的用户名为 xiaoming，那么项目名称就填 xiaoming.github.io。

创建完成后，点击设置，拉到下方 GitHub Pages 部分，点击链接即可看到 GitHub Pages 的初始界面。把这个地址记下来，以后就是自己博客的地址了。

Github Desktop

对于初次使用 GitHub 的小白用户来讲，Github Desktop 是一个比较简单的选择。

首先在 官方网站 进行下载，只需选择自己电脑的操作系统，按照安装提示一步一步操作，即可完成安装。

安装完成后，打开 Github Desktop，登录自己的账号。如下图所示，先选择 Clone repository，再选择刚才创建的项目库，为其指定一个地址，点击 Clone 即可将其克隆至本地。

对于中国大陆地区用户，可以在安装完成后为 Github 配置代理。

克隆完成后，就可以在刚才指定的地址看到一个名为 username.github.io 的文件夹。此时把储存网站的文件夹里 public 文件夹中的所有文件复制进来，再打开 Github Desktop，会发现提示项目库有更改。选择 Commit to master 将更改合并到主分支，再选择 Push origin 进行推送，即可完成网站的部署。

Git

Git 是一个免费的开源分布式版本控制系统，旨在快速高效地处理从小型到大型项目的所有内容。Git 可以在其官方的 下载页面 进行下载，同样只需选择自己电脑的操作系统，按照安装提示一步一步操作，即可完成安装。
